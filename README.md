## The Logical Learning Path (Syllabus)

To build a strong foundation, you must learn these topics in this exact order. Each topic directly builds upon the previous one!

### Phase 1: The Absolute Fundamentals (Days 1-5)
1. **Python Fast I/O & Tricks**: How to read inputs faster than `input()`, list comprehensions, slicing. *(Prerequisite for everything)*
2. **Time & Space Complexity (Big O)**: How to know if your code will pass or get a Time Limit Exceeded (TLE).
3. **Basic Number Theory**: Prime numbers (Sieve of Eratosthenes), GCD, LCM, Modulo Arithmetic. *(High probability in TCS Aptitude and Coding)*

### Phase 2: Mastering 1D Data (Days 6-12)
4. **Arrays & Strings**: Traversals, reversing, palindrome checks, basic manipulation.
5. **Hashing (Dictionaries/Sets)**: Frequency arrays, finding duplicates, O(1) lookups. *(Crucial for optimizing O(N^2) to O(N))*
6. **Two Pointers & Sliding Window**: Finding pairs, longest substrings, subarrays. *(Uses Arrays + Hashing concepts)*
7. **Prefix Sums**: Fast range sum queries.

### Phase 3: Matrix & Search (Days 13-18)
8. **2D Arrays (Matrices)**: Row/Col traversals, diagonals, matrix rotation.
9. **Binary Search**: Not just on arrays, but "Binary Search on Answer" (Advanced).
10. **Sorting Algorithms**: Merge Sort, Quick Sort concepts, and when to use Python's `sort()` vs custom comparators.

### Phase 4: Non-Linear & Advanced DS (Days 19-25)
11. **Basic Recursion**: Function calling itself. Understanding the call stack.
12. **Stacks & Queues**: Next Greater Element, balanced parentheses. *(Uses Arrays/Linked Lists)*
13. **Linked Lists**: Node traversal, reversing, Floyd's Cycle Detection.
14. **Trees (Binary Trees & BST)**: DFS, BFS (Level Order), views of a tree. *(Heavy recursion usage)*

### Phase 5: The Final Bosses (Days 26-30)
15. **Graphs (Basics)**: Representing graphs (Adjacency List using Hashmaps), BFS, DFS.
16. **Dynamic Programming (1D)**: Fibonacci, Climbing Stairs, Knapsack. *(Uses Recursion + Hashing/Memoization)*
17. **TCS NQT PYQ Marathon**: Solving the hardest previous year questions using the optimal techniques learned above.
